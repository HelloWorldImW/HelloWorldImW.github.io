iOS开发中遇到的错误

发表于:2016-12-26，更新于:2018-01-03，By Darren
大纲
  • 错误描述： Failed to chmod user/Library/Developer/CoreSimulator/Devices NO Such File Or directory
    原因： 在工程删除文件时,没有把文件的映射删除掉(一般都会自动删除)
    解决方法： 查找AddedLib Resources complied Resources中是否还存在已经删除文件的映射并删除
    引用：StackOverFlow中Vikas Mishra的回答
  • 错误描述：directory not found for option '-F/Applications/···
    原因：工程文件中Build Settings -> Search Paths键的值有冲突
    解决方法：将老的(引起冲突的)sarch path删除掉
    引用：StackOverFlow中King-Wizard的回答
